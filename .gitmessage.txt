[Type] title (#이슈번호)
# [Header]
# 형식 - [Type] title (#이슈번호)
#
# [Type] 입력 목록
#  1. ✨ Feat: 새로운 기능 추가
#  2. 🐛 Fix: 오류 해결
#  3. 🚑 Hotfix: 긴급 수정
#  4. ♻️ Refactor: 코드 리팩터링
#  5. ✅ Test: 테스트와 관련된 모든 것
#  6. 📝 Docs: 문서화에 관한 모든 것
#  7. 💡 Chore: 오타 수정 및 새로운 기능이 추가되지 않고, 코드가 변경 된 경우 (주석 추가 및 수정 포함)
#
# [Title] 규칙
#  1. 50자 이내 짧은 요약 (영어의 경우 첫글자는 대문자로)
#  2. 명령형으로 작성 (과거형X)
#  3. 제목 끝에 .(마침표) 금지
#
# [Body] 작성 규칙
#  1. 제목과 본문 사이 빈 칸 한줄 필수 (제목과 본문 분리)
#  2. 생략 가능, 필요 시 자세한 설명 작성 (72자 내외로 마무리)
#  3. 적용한 변경사항과 그것을 만든 이유에 대해 설명
#     - 어떻게보다는 "무엇을", "왜"
